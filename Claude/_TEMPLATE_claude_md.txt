# Instruções para Claude - Projeto DR.X Xjur

## 📋 O Que é Este Projeto

**DR.X** é um ecossistema jurídico SaaS em produção que automatiza fluxos para escritórios de advocacia.

- **Status**: Production Ready (v1.0.001)
- **Tipo**: Monorepo (Turborepo)
- **Stack**: NestJS (API) + React (Web) + PostgreSQL + Redis
- **Modelo**: Multi-tenant (cada cliente isolado)
- **Proprietário**: DR.X / Evandro Xavier

## 🏗️ Arquitetura Técnica

### Monorepo Structure
```
apps/
  ├── api/        # NestJS (TypeScript) - Backend REST
  ├── web/        # React (TypeScript) - Frontend SPA
  ├── extension/  # Chrome Extension (React)
  └── storage/    # Storage Service
```

## ⚙️ Padrões Obrigatórios

### 1. Estrutura NestJS (Backend)

Sempre siga:
```
src/[feature]/
├── [feature].service.ts      # Lógica de negócio
├── [feature].controller.ts   # Endpoints HTTP
├── [feature].module.ts       # Declaração NestJS
├── dto/
│   ├── create-[feature].dto.ts
│   └── update-[feature].dto.ts
└── interfaces/
    └── [feature].interface.ts
```

Regra: Lógica em Service, endpoints em Controller.

### 2. Multi-Tenancy (Obrigatório)

**TODA** query de dados de negócio precisa do `tenantId`:

```typescript
// ❌ ERRADO
async getContacts() {
  return this.prisma.contact.findMany();
}

// ✅ CORRETO
async getContacts(tenantId: string) {
  return this.prisma.contact.findMany({
    where: { tenantId }
  });
}
```

### 3. Prisma Schema

Local: `packages/database/prisma/schema.prisma`

```prisma
model Contact {
  id       String  @id @default(cuid())
  tenantId String  // Sempre adicione isto
  name     String
  // ... outros campos

  tenant   Tenant  @relation(fields: [tenantId], references: [id])
}
```

Depois de editar schema:
```bash
npx prisma migrate dev --name "descricao da mudanca"
git add packages/database/prisma/migrations/
git commit -m "chore: add migration for X"
```

### 4. DTOs e Validação

```typescript
import { IsString, IsEmail, IsOptional } from 'class-validator';

export class CreateContactDto {
  @IsString()
  name: string;

  @IsEmail()
  email: string;

  @IsOptional()
  @IsString()
  phone?: string;
}
```

## 🤖 Sistema de Skills (IA/Automação)

### Tipos
- **SYSTEM Skills**: Produto oficial (triagem, cobranca, agenda, analise processual)
- **CUSTOM Skills**: Específico de cada cliente (criado/removido por eles)

### Como Implementar Nova Skill

1. Crie em: `src/agent/skills/[nome-skill]/`
2. Implemente `execute()` method
3. Registre em: `src/agent/agent.service.ts`
4. Configure em UI: `apps/web/components/skills/`
5. Teste em: `Settings > Skills > Playground`
6. Defina: SYSTEM ou CUSTOM

### Quando Sugerir
Se a feature envolve análise de texto/IA, faça como skill em vez de endpoint.

## 📞 Integrações Esperadas

### WhatsApp (Evolution API)
- Config: `src/connections/evolution/`
- Webhook: `/api/evolution/webhook`
- ENV vars: `EVO_API_KEY`, `EVO_SERVER_URL`, `EVO_WEBHOOK_URL`

**Conectividade**:
- Local: `http://localhost:8080` (Evolution) → `http://host.docker.internal:3000/api/...` (webhook)
- Produção: `http://evolution:8080` (Evolution) → `http://api:3000/api/...` (webhook)

### Microsoft 365
- Config: `src/connections/microsoft/`
- Tokens: Armazenados criptografados em DB
- Sync: Calendário, documentos

### OpenAI
- Config: `src/agent/openai/`
- ENV: `OPENAI_API_KEY`
- Uso: Análise + sugestões em skills

## ⚠️ "Gotchas" Evitar

1. **Multi-tenancy bypass**: Esqueceu tenantId em query → data leak
2. **Deletou skill SYSTEM**: Sempre PAUSE, nunca DELETE
3. **Schema sem migração**: Adiciona coluna mas não roda `npx prisma migrate dev`
4. **Breaking API changes**: Não versiona endpoints (`/api/v2/`)
5. **Prod DB em localhost**: Edita producao acidentalmente
6. **Evolution webhook errado**: Local precisa `host.docker.internal`
